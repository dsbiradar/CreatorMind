## 🌌 What is CreatorMind AI?

**CreatorMind AI** is a next-generation, AI-powered **Creator Operating System** — a single, unified platform engineered for content creators, influencers, digital marketers, and brand strategists who refuse to be average.

Forget juggling 15 different tools. CreatorMind AI brings **everything** into one futuristic workspace:

- 🧠 **Generate** scroll-stopping content from a mood, a voice, or a single idea
- 📈 **Predict** whether your content will go viral — before you post it
- 🔍 **Analyze** trending topics and audience signals in real time
- ♻️ **Repurpose** one piece of content into 10+ platform-ready formats
- 🎨 **Create** stunning AI-generated visuals and video scripts on demand
- 🚀 **Optimize** your entire social growth strategy with AI intelligence

> *CreatorMind AI doesn't just assist creators — it thinks like one.*

## ✨ Key Features

### 🎭 Mood-to-Content AI
> *Turn a feeling into a viral post*

Describe your mood — frustrated, hyped, nostalgic, inspired — and CreatorMind AI reverse-engineers the perfect content angle, tone, and hook for any platform. Emotional intelligence meets content strategy.

---

### 🎙️ Voice-to-Content AI
> *Speak your ideas. Publish polished content.*

Record or upload a voice note and watch as CreatorMind AI transcribes, structures, and transforms your raw thoughts into platform-optimized posts, threads, scripts, or newsletters — in seconds.

### 🧑‍⚖️ AI Judge My Content
> *Brutal honesty. Better results.*

Submit any draft and get an AI-powered content audit — rated on hook strength, emotional pull, readability, platform fit, and virality potential. Includes actionable rewrite suggestions.

### 🔮 Viral Prediction Engine
> *Know before you post*

Powered by trend analysis and engagement modeling, the Viral Prediction Engine scores your content on its probability to go viral — broken down by platform, audience segment, and time of post.

---

### ♻️ Repurpose Studio
> *One idea. Infinite formats.*

Transform a single blog post, YouTube script, or tweet into Instagram carousels, LinkedIn articles, TikTok scripts, email newsletters, Pinterest pins, and more — all formatted natively for each platform.

---

### 📡 Trend Finder
> *Stay ahead of the algorithm*

Real-time trend radar that surfaces rising hashtags, viral sounds, emerging topics, and competitor content strategies — filtered by niche, platform, and geography. Never miss a trend wave again.

---

### 🗓️ Smart Content Planner
> *Plan smarter. Post with purpose.*

AI-assisted content calendar that schedules your posts at peak engagement windows, maintains a consistent brand voice, and auto-fills your pipeline based on trending opportunities.

---

### 🌍 Local Language / Global Creator Mode

Generate, translate, and adapt content across 50+ languages while preserving cultural nuance, regional slang, and local platform behavior. Go global without losing your authentic voice.

---

### 🖼️ AI Image Studio
> *Your visual identity. Instantly generated.*

Design thumbnails, social media graphics, carousels, and brand assets with text-to-image AI. Apply your brand colors, fonts, and style presets for a consistent, professional look across all content.

---

### 🎬 AI Video Studio
> *Scripts. Storyboards. Ready to shoot.*

Generate full video scripts, scene-by-scene storyboards, B-roll suggestions, and caption overlays for YouTube, Reels, TikTok, and Shorts — powered by your niche and audience data.

---

### 📲 Multiplatform Content Adapter
> *One click. Every platform optimized.*

Automatically reformat content for the unique specs, character limits, aspect ratios, and tone requirements of Instagram, X (Twitter), LinkedIn, YouTube, TikTok, Pinterest, and more.

## 🚀 Getting Started

### Prerequisites

- Node.js `>= 18.x`
- npm or yarn
- Google Gemini API key
- Firebase or Supabase project

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/OMKAR-tech-hub/creatormind-ai.git

# 2. Navigate into the project
cd creatormind-ai

# 3. Install dependencies
npm install

# 4. Set up your environment variables
cp .env.example .env.local

# 5. Start the development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser. 🎉

---

## 🔐 Environment Variables

Create a `.env.local` file in the root directory. Use `.env.example` as your reference:

```env
# ─────────────────────────────────────────
# 🤖 AI ENGINE
# ─────────────────────────────────────────
GEMINI_API_KEY=your_gemini_api_key_here

# ─────────────────────────────────────────
# 🔥 FIREBASE CONFIGURATION
# ─────────────────────────────────────────
NEXT_PUBLIC_FIREBASE_API_KEY=your_firebase_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id

# ─────────────────────────────────────────
# 🟢 SUPABASE CONFIGURATION
# ─────────────────────────────────────────
NEXT_PUBLIC_SUPABASE_URL=https://your_project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# ─────────────────────────────────────────
# ⚙️ APP CONFIG
# ─────────────────────────────────────────
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_APP_ENV=development
```

## 🤝 Contributing

We welcome contributions from the community! Whether it's a bug fix, a new feature, or improved documentation — every PR matters.

### How to Contribute

```bash
# 1. Fork the repository
# 2. Create your feature branch
git checkout -b feature/your-feature-name

# 3. Make your changes and commit
git commit -m "feat: add your feature description"

# 4. Push to your branch
git push origin feature/your-feature-name

# 5. Open a Pull Request on GitHub
```

### Contribution Guidelines

- Follow the existing code style and TypeScript conventions
- Write meaningful commit messages (we use [Conventional Commits](https://www.conventionalcommits.org/))
- Add comments for complex logic
- Test your changes before submitting a PR
- Keep PRs focused — one feature or fix per pull request
- Open an issue first for major feature proposals

### Found a Bug? 🐛

Open an issue with the `bug` label and include:
- A clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
